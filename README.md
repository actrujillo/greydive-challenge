# greydive-challenge

En el formulario utilice el JSON directo porque considere que son datos estaticos, de esta manera simplificar la tarea de la aplicacion. 
Luego, accedi mediante una constante a los items y desde ahi comence a trabajar con los datos.

Inicialize Firebase desde el componente credenciales para poder tener acceso desde los dos componentes que lo requerian: el formulario y los usuarios.

En el Header me atrevi a utilizar una imagen de su empresa porque considere que daba mejor aspecto y, de esta manera, evitaba poner un titulo generico como "ingresar datos".
Ese logo lo introduje desde un link y no desde una imagen descargada para una aplicacion mas liviana.
En el mismo Header no se encuentra un archivo css importado debido a su casi nulo estilo.

En la parte de los usuarios utilice un span en las etiquetas de los datos traidos de la Firesbase para no tocar demasiado el codigo y tener una solucion simple y buscada.
